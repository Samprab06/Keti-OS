# Contributing to Keti OS
 
Keti OS is open source and free to use. If you fork it, a mention or link back to the original repo is appreciated (though not legally required). Built by Aadithya Vasudevan (Aadithya-19).
 
---
 
## How to contribute
 
- Fork the repo
- Pick an open issue or suggest a feature
- Submit a pull request with a clear description of what you built and why
If you're not sure where to start, check the contribution tiers in the README - most contributors will never need to touch assembly.
 
---
 
## Ground rules
 
- **Understand your own code.** No blatant copy-paste - if you submit something, be ready to explain what it does and why. This keeps the codebase readable for everyone.
- **Document it.** Add comments to non-obvious functions. Future contributors (and you, six months later) will thank you. I hate comments too, but its helpful.
- **Log it.** Add a folder under `docs/` for what you worked on - design notes, debugging stories, dated entries. See the README for the format.
- **Cite your resources.** If a blog post, paper, or video helped you, mention it in your log or PR description. We can add it to the resource board. Even if you use any AI, give it a shoutout. You never know...
- **Keep it fun.** This is a learning project first. Don't stress perfection - working and understanding beats perfect and opaque.
